## Affected Pages/Routes
- Feishu event ingress (`im.message.receive_v1`) — verify text parsing, mention filtering, auth allowlist, duplicate suppression.
- Feishu card actions (`card.action.trigger`) — verify `/abort`, `/retry`, `/choose`, `/reset` action mapping and idempotency.
- Health endpoint (`GET /healthz`) — verify payload shape, adapter transport source, and null-normalized metrics.

## Key Interactions to Verify
- User sends task in private chat: enqueue -> run -> completion message/card -> conversation state update.
- User sends task in group chat without bind: receives bind prompt and no task enqueue.
- Non-resumable provider task: no session reuse, no context compaction, conversation session remains empty.
- Interaction flow: model returns interaction block -> pending interaction persisted -> `/choose` resumes follow-up task.

## Edge Cases
- Malformed message payload JSON in Feishu event.
- Duplicate message IDs and duplicate task text within duplicate window.
- Queue/chat/user limits reached.
- Abort during run and abort during post-run with auto-commit rollback behavior.
- Missing channel metrics in health payload.

## Critical Paths
- Event -> dispatch -> command/task routing -> runtime queue -> provider execution -> channel updates.
- Task execution finalization path for success/failed/cancelled/interrupted.
- State persistence + recovery path (`queue`, `running`, `interrupted`) across restart.
